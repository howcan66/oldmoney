=============================================================
PALETTE MIGRATION RULES
=============================================================

Switching from Modern to Old Money Theme:
1. Update CSS variables in style.css
2. Replace all Modern Theme hex codes with Old Money equivalents
3. Test all pages for visual consistency
4. Verify contrast ratios meet WCAG AA standards

Rollback Procedure:
1. Revert CSS variable changes
2. Restore Modern Theme hex codes
3. Clear browser cache
4. Test all interactive elements

=============================================================
USAGE GUIDELINES
=============================================================

DO:
• Use semantic color names in CSS (--color-primary, --color-success)
• Maintain consistent contrast ratios (4.5:1 minimum for body text)
• Test colors in both light and dark environments
• Document any new color additions in this file first

DON'T:
• Hard-code hex values directly in HTML
• Mix colors from Modern and Old Money themes
• Use colors not defined in this palette
• Override semantic colors for decorative purposes

=============================================================
COLOR ACCESSIBILITY
=============================================================

All color combinations in both themes meet WCAG 2.1 Level AA:
• Normal text: 4.5:1 contrast ratio minimum
• Large text (18pt+): 3:1 contrast ratio minimum
• Interactive elements: clearly distinguishable states

Test tools:
• WebAIM Contrast Checker
• Chrome DevTools Accessibility Panel
• axe DevTools browser extension
