ARCHITECTURE
=============================================================

PORTAL PAGE STRUCTURE:
- Header: Site title, subtitle, branding
- Category Sections: Grouped applications by type/purpose
- App Cards: Individual application entries with descriptions
- Footer: Copyright, timestamp, metadata

DESIGN PRINCIPLES:
- Clean, modern, responsive layout
- Card-based design for easy scanning
- Visual hierarchy with icons/emojis
- Gradient backgrounds for visual appeal
- Hover effects for interactivity
- Mobile-first responsive design

LINKING STRATEGY:
- Public URLs: GitHub Pages links (https://username.github.io/repo/page.html)
- Local URLs: Direct file paths (file:///C:/Path/to/file.html)
- Both options provided for flexibility
- Target="_blank" for external navigation
- Security attributes on all external links

=============================================================
SECURITY: rel="noopener noreferrer"
=============================================================

WHY IT'S CRITICAL:
When using target="_blank", opened pages can access window.opener
and potentially:
1. Redirect the parent page (phishing attack)
2. Read parent page properties
3. Execute malicious JavaScript

SOLUTION:
Always add rel="noopener noreferrer" to target="_blank" links:

<a href="https://external-site.com" 
   target="_blank" 
   rel="noopener noreferrer">

WHAT IT DOES:
- noopener: Prevents window.opener access
- noreferrer: Doesn't send referrer information

WHEN TO USE:
- ALL external links with target="_blank"
- Even links to your own GitHub Pages repos
- Defense-in-depth security practice

GITHUB PAGES DEPLOYMENT
=============================================================

PREREQUISITES:
1. GitHub repository exists
2. HTML file committed to repo
3. Repository is public (or GitHub Pro for private)

SETUP STEPS:
1. Go to: https://github.com/username/repo-name/settings/pages
2. Source: Select branch (main, master, or Main-Test)
3. Folder: Select / (root) or /docs
4. Click Save
5. Wait 1-2 minutes for build

URL PATTERN:
https://username.github.io/repo-name/
https://username.github.io/repo-name/page.html

CUSTOM DOMAIN (Optional):
1. Add CNAME file to repo root
2. Configure DNS records
3. Enable HTTPS in GitHub Pages settings

TROUBLESHOOTING:
- 404 Error: Check branch/folder settings
- Build Failed: Check HTML syntax, no Jekyll errors
- Changes Not Visible: Clear browser cache, wait for rebuild
- HTTPS Issues: Enforce HTTPS in settings

=============================================================
COMMON ISSUES & SOLUTIONS
=============================================================

ISSUE: Cards not aligning in grid
SOLUTION: Check minmax() values, ensure all cards have same structure

ISSUE: Gradient not visible on mobile
SOLUTION: Set min-height: 100vh on body element

ISSUE: Links open in same tab
SOLUTION: Add target="_blank" to all external links

ISSUE: Favicon not showing
SOLUTION: Clear browser cache, check data URI encoding

ISSUE: GitHub Pages 404 error
SOLUTION: Verify branch settings, check file path capitalization

ISSUE: Hover effects laggy
SOLUTION: Use transform instead of position changes, add will-change: transform

ISSUE: Text unreadable on gradient
SOLUTION: Increase contrast, add text-shadow, use semi-transparent overlays

ISSUE: Layout breaks on small screens
SOLUTION: Add @media query for mobile, use grid-template-columns: 1fr
